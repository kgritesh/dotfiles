# LEARNINGS.md Template

Use this template when starting significant work that will accumulate learnings over time.
This file is temporary — it gets merged into CLAUDE.md at the end of the feature and then deleted.

```markdown
# Learnings: [Feature/Task Name]

## Gotchas

### [Descriptive title]
- **Context**: When this occurs
- **Issue**: What goes wrong
- **Solution**: How to handle it

## Patterns

### [Pattern name]
- **What**: Brief description
- **Why it works**: Rationale
- **Example**: Code snippet if helpful

## Decisions

### [Decision title]
- **Options**: What was considered
- **Chosen**: What was picked
- **Rationale**: Why
- **Trade-off**: What was given up

## Edge Cases

- [Description]: How it was handled
```

## Guidelines

- Add entries as you discover them — don't wait until the end
- Keep entries informal during capture; polish happens at merge time
- Not everything in LEARNINGS.md will make it into CLAUDE.md — that's fine
- The merge step filters for significance and deduplicates
